<h1>03 – Outbound Outreach (LinkedIn + Email)</h1>

<hr/>

<h2>1. Overview</h2>

<p>
The <strong>03–Outbound Outreach</strong> workflow is the execution layer of the AI Outreach Automation system.
It is responsible for <strong>delivering AI-generated messages to real prospects</strong> across multiple outbound channels
while maintaining strict control over sequencing, tracking, and human-like behavior.
</p>

<p>
This workflow consumes the enriched and personalized lead data produced by
<strong>02–AI Personalization</strong> and performs <strong>multi-channel outbound engagement</strong> using:
</p>

<ul>
  <li>LinkedIn connection invitations</li>
  <li>Cold email outreach</li>
</ul>

<p>
Both channels are executed as part of a <strong>single, atomic outbound cycle per lead</strong>.
</p>

<h2>3. Why LinkedIn and Email Are Combined</h2>

<p>
LinkedIn and Email outreach are implemented as a <strong>single workflow</strong> because:
</p>

<ul>
  <li>They operate on the same lead record</li>
  <li>They share execution order and timing logic</li>
  <li>They use the same tracking and persistence layer</li>
  <li>They must remain synchronized for accurate reporting</li>
</ul>

<p>
Separating them would introduce state fragmentation and execution inconsistency.
</p>

<h2>6. Trigger & Inputs</h2>

<h3>6.1 Trigger Type</h3>
<ul>
  <li>Workflow-to-workflow execution trigger</li>
</ul>

<h3>6.2 Input Parameters</h3>
<ul>
  <li>Spreadsheet ID</li>
  <li>Sheet ID</li>
</ul>

<p>
These parameters reference the campaign-specific Google Sheet created during lead acquisition.
</p>

<hr/>

<h2>7. Input Data Contract</h2>

<p>
Each lead record must include:
</p>

<ul>
  <li>Unique lead ID</li>
  <li>LinkedIn profile URL</li>
  <li>LinkedIn provider ID</li>
  <li>Email address</li>
  <li>AI-generated LinkedIn icebreaker</li>
  <li>AI-generated email subject</li>
  <li>AI-generated email body</li>
</ul>

<p>
Leads missing any mandatory field are excluded from execution.
</p>

<hr/>

<h2>8. Sequential Execution Flow</h2>

<h3>Step 1: Data Retrieval</h3>
<ul>
  <li>Fetches enriched lead records from Google Sheets</li>
  <li>Data is processed in controlled batches</li>
</ul>

<h3>Step 2: Lead Iteration</h3>
<ul>
  <li>Each lead is processed independently</li>
  <li>Ensures isolation and failure containment</li>
</ul>

<h3>Step 3: LinkedIn Invitation Dispatch</h3>
<ul>
  <li>Unipile API sends a LinkedIn connection request</li>
  <li>AI-generated icebreaker included</li>
  <li>Provider ID used for deterministic routing</li>
</ul>

<h3>Step 4: LinkedIn Invite Status Update</h3>
<ul>
  <li>Invitation timestamp written to Google Sheets</li>
  <li>Status fields updated for tracking</li>
</ul>

<h3>Step 5: LinkedIn Invite History Logging</h3>
<ul>
  <li>Session ID and icebreaker stored in PostgreSQL</li>
  <li>Used later for conversation continuity</li>
</ul>

<h3>Step 6: Email Dispatch</h3>
<ul>
  <li>Email sent via Gmail integration</li>
  <li>Subject and body generated by AI</li>
  <li>Executed after LinkedIn invite logic</li>
</ul>

<h3>Step 7: Email Status Logging</h3>
<ul>
  <li>Email sent flag updated in Google Sheets</li>
  <li>Supports reporting and follow-ups</li>
</ul>

<h3>Step 8: Randomized Delay Injection</h3>
<ul>
  <li>Random delay (0–7 seconds) applied</li>
  <li>Prevents automation pattern detection</li>
</ul>

<hr/>

<h2>9. Data Persistence & Tracking</h2>

<h3>9.1 Google Sheets</h3>
<ul>
  <li>Primary operational datastore</li>
  <li>Tracks:
    <ul>
      <li>LinkedIn invite sent</li>
      <li>Email sent</li>
      <li>Timestamps</li>
    </ul>
  </li>
</ul>

<h3>9.2 PostgreSQL</h3>
<ul>
  <li>Conversation session initialization</li>
  <li>Used by downstream AI reply workflows</li>
</ul>

<hr/>

<h2>10. Failure Handling & Safeguards</h2>

<ul>
  <li>Invalid provider IDs are skipped</li>
  <li>API failures do not halt entire batch</li>
  <li>Partial executions are traceable</li>
  <li>No duplicate invites sent</li>
</ul>

<hr/>

<h2>11. Rate Limiting & Anti-Detection Controls</h2>

<ul>
  <li>Batch-based execution</li>
  <li>Randomized wait times</li>
  <li>Sequential channel dispatch</li>
  <li>No parallel LinkedIn actions</li>
</ul>
